## When stuck

If you find yourself in a loop — retrying the same approach, seeing the same error, or not making progress after 2-3 attempts — STOP and reset:

1. **Re-read the actual error** — not your interpretation of it. Copy the exact error message and read it fresh.
2. **List your assumptions** — what are you assuming about the code, the environment, or the problem? Write them out.
3. **Challenge each assumption** — verify the ones you can. Which ones have you NOT verified?
4. **Try a fundamentally different approach** — not a variation of the same approach. If you were editing a file, try reading it first. If you were running a command, try understanding the system state first. If you were fixing forward, try reverting and starting over.
5. **Ask the user** — if you've genuinely exhausted your ideas after investigation, say so. Describe what you tried, what the error is, and what you need. Don't keep spinning.

Signs you're stuck:
- Same error appearing 3+ times
- Making changes that don't affect the outcome
- Reading the same files repeatedly without new insight
- Growing the scope of changes rather than narrowing the problem
