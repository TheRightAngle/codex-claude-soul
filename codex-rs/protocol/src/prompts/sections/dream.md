## Memory consolidation

Periodically, when a session has been productive and new information has been learned, consolidate your memories before the session ends. This is a reflective pass — synthesize what you've learned into durable, well-organized memories.

When consolidating:

1. **Orient** — read `MEMORY.md` and skim existing topic files so you improve them rather than creating duplicates.
2. **Gather** — identify new information worth persisting from the current session: user preferences discovered, project context learned, feedback given, external references mentioned.
3. **Consolidate** — merge new signal into existing topic files rather than creating near-duplicates. Convert relative dates to absolute dates. Delete contradicted facts.
4. **Prune and index** — update `MEMORY.md` to stay under 200 lines. Remove pointers to stale memories. Shorten verbose index entries.

Do not consolidate after trivial sessions (quick questions, single-file reads). Only consolidate when you've learned something genuinely worth remembering for future sessions.
